> เอกสารอธิบายโครงสร้างข้อมูลใน Google Sheet `case_data`
> **สำหรับคนที่มารับช่วงดูแลระบบต่อ** — อ่านไฟล์นี้ก่อนจะเข้าใจข้อมูลทั้งหมด

> - Frontend: `index.html` (GitHub Pages) — ไม่รู้จักโครงสร้างแท็บ คุยผ่าน JSON กับ backend เท่านั้น
> - อัปเดตล่าสุด: 2026-07-03

## หลักการ v3

- **1 แท็บ = 1 เรื่อง, 1 แถว = 1 รายการ** — ไม่มี JSON ยัดในเซลล์ ไม่มีข้อมูลซ้ำสองที่
- ทุกแท็บเชื่อมกันด้วยคอลัมน์ **เลขที่รายงาน** (เช่น `INC-260702-0001`)
- อยากรู้ข้อมูลอะไรของคดีไหน → เปิดแท็บนั้น แล้ว Filter ด้วยเลขที่รายงาน
- ค่าในตารางเป็นภาษาไทยให้อ่านรู้เรื่อง (backend แปลงกลับเป็นค่าระบบตอนส่งให้หน้าเว็บ)

## ผังแท็บ

```
📁 case_data
├── คดี               ← ตารางหลัก 1 แถว = 1 คดี
├── ผู้เสียหาย          ← 1 แถว = ผู้เสียหาย 1 คน
├── ผู้ต้องหา           ← 1 แถว = ผู้ต้องหา 1 คน
├── ทรัพย์สินเสียหาย    ← 1 แถว = 1 รายการ
├── ของกลาง            ← 1 แถว = 1 ชิ้น (แยกจากทรัพย์สิน เพื่อสถิติ)
├── อาวุธ              ← 1 แถว = 1 ชิ้น
├── ไฟล์แนบ            ← 1 แถว = 1 ไฟล์ (ตัวไฟล์จริงอยู่ Google Drive)
├── officer            ← เจ้าหน้าที่ + เบอร์ + PIN (login)
├── station            ← รายชื่อ สภ. (auto-learn)
├── _คำอธิบาย          ← สรุปเอกสารนี้ ฝังใน Sheet
└── case               ← 🔴 archive ข้อมูลเก่า v2 — ห้ามลบ, ไม่มีการเขียนเพิ่ม
```

## กติกาการดูแลรักษา

1. **ห้ามเปลี่ยน format คอลัมน์ที่เป็น Plain text** (วันที่/เวลา/โทร/เลขบัตร) — ถ้าเปลี่ยนเป็นอัตโนมัติ Sheets จะแปลงค่าแล้วข้อมูลเพี้ยน
2. **เพิ่มคอลัมน์ใหม่ = ต่อท้ายเท่านั้น** และแก้ `H_*` + `*RowFromData` + `readAllRecords` ใน `apps_script_v3.gs` ให้ลำดับตรงกันเสมอ
3. **ห้ามลบแท็บ `case`** — เป็น archive ของข้อมูลก่อนย้ายระบบ (มิ.ย.–ก.ค. 2569, 23 คดี)
4. เลขที่รายงานซ้ำ: backend กันให้อัตโนมัติ (LockService + `uniqueRef()`)
5. รหัสลับเก็บใน **Script Properties**: `ADMIN_SECRET`, `LINE_TOKEN`, `LINE_GROUP_ID`

## การ Deploy

1. แก้ `apps_script_v3.gs` → วางทับใน Apps Script Editor → Save
2. ครั้งแรกเท่านั้น: รัน `setupV3()` แล้ว `migrateV3()` จาก editor
3. Deploy → Manage deployments → ✏️ **แก้ deployment เดิม** → New version → Deploy (URL เดิมต้องไม่เปลี่ยน)
4. Frontend: แก้ `index_v2.html` → `cp index_v2.html index.html` → commit + push (GitHub Pages)

## ภาคผนวก: แท็บ `case` เดิม (archive, 44 คอลัมน์)

โครงเก่ายัดทุกอย่างในแถวเดียว: คอลัมน์แบน A–AJ + JSON 5 ช่อง (victims_json, perps_json, damages_json, evidence_json, weapons_json) + URL ไฟล์คั่นด้วย `|`
ผู้เสียหาย/ผู้ก่อเหตุ "คนที่ 1" ถูกเก็บซ้ำ 2 ที่ (คอลัมน์แบน + JSON) — **ยึด JSON เป็นหลัก**
คอลัมน์ อาวุธ/ชนิดอาวุธ/รายละเอียดอาวุธ (AC–AE) เป็น legacy ก่อนมี weapons_json
ดูรายละเอียดเต็มได้จากโค้ด `migrateV3()` ซึ่ง map ทุกคอลัมน์เก่า → แท็บใหม่
